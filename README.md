# Алиса. Доступное Обучение

Платформа адаптивного учебного тестирования для людей с ограниченными возможностями здоровья (ОВЗ), построенная на базе Алиса AI и Yandex AI Studio.

## О проекте

Сервис позволяет создавать и проходить учебные тесты, автоматически адаптированные под потребности пользователей с различными формами ОВЗ:

- **Слабовидение** — аудио-описания изображений, голосовой интерфейс
- **Нарушения ОДА** — сокращённое количество вариантов, голосовое управление
- **Когнитивные особенности** — упрощённые формулировки, аналогии, пошаговые подсказки

### Ключевые возможности

- 🎯 Создание тестов с автоматической адаптацией под тип ОВЗ
- 🔗 Публичные тесты с возможностью делиться ссылкой
- 🗣️ Голосовой интерфейс через Алиса AI
- 👁️ Голосовой помощник на фронтенде для слабовидящих
- 🧠 RAG-подсказки из базы знаний инклюзивного образования
- 📊 Сохранение результатов прохождения тестов
- 🌙 Светлая и тёмная тема

### Использованные возможности Алиса AI / Yandex AI Studio

| Возможность | Применение |
|:---|:---|
| YandexGPT | Адаптация текста вопросов под тип ОВЗ |
| SpeechKit (TTS/ASR) | Озвучивание вопросов, голосовой ввод ответов |
| Vision (OCR) | Извлечение текста из загруженных PDF/DOCX |
| RAG | Поиск подсказок в базе знаний инклюзивного образования |
| Function Calling | Вызов функций адаптации, сохранения результатов |
| Персонализация | Адаптация интерфейса под профиль пользователя |
| Голосовые сценарии | Полный голосовой диалог при прохождении теста |

## ⚠️ Обязательные зависимости

Приложение **не запустится**, если любая из зависимостей недоступна. При старте выполняется строгая проверка:

| Зависимость | Переменные окружения | Обязательность | Описание |
|:---|:---|:---|:---|
| **Qdrant** | `QDRANT_URL` | Обязательно | Векторная БД для RAG. Должна быть запущена и доступна по сети. |
| **Yandex API** | `YANDEX_API_KEY`, `YANDEX_FOLDER_ID` | Обязательно | API-ключ для YandexGPT, SpeechKit, Vision. |
| **Алиса** | `ALICE_SKILL_ID`, `ALICE_OAUTH_TOKEN` | Опционально | Для управления навыком через API Яндекс.Диалогов. Вебхук работает и без них. |

При отсутствии обязательных зависимостей приложение выведет подробную ошибку и завершится с кодом 1.

## Быстрый старт

### Предварительные требования

- Python 3.11+
- Node.js 18+ и npm
- **Docker и Docker Compose** (рекомендуется)
- Аккаунт в [Yandex Cloud](https://console.yandex.cloud/) с включёнными API
- (Опционально) Навык в [Яндекс.Диалогах](https://dialogs.yandex.ru/) для полной интеграции с Алисой

### Вариант A: Запуск через Docker Compose (рекомендуется)

#### 1. Клонирование репозитория

```bash
git clone <repo-url>
cd Alice_Hackaton
```

#### 2. Настройка переменных окружения

```bash
cp .env.example .env
```

Отредактируйте `.env`, заполнив **все обязательные** поля:

```env
# ─── Yandex Cloud (ОБЯЗАТЕЛЬНО) ───
YANDEX_API_KEY=your_real_api_key
YANDEX_FOLDER_ID=your_real_folder_id
YANDEX_GPT_MODEL=yandexgpt-lite

# ─── Qdrant (ОБЯЗАТЕЛЬНО) ───
QDRANT_URL=http://qdrant:6333
QDRANT_COLLECTION_NAME=knowledge_base
QDRANT_API_KEY=

# ─── Алиса (опционально) ───
ALICE_SKILL_ID=
ALICE_OAUTH_TOKEN=
```

> **Важно:** При запуске через Docker Compose `QDRANT_URL` должен быть `http://qdrant:6333` (имя сервиса в Docker), а не `http://localhost:6333`.

#### 3. Запуск всех сервисов

```bash
docker-compose up --build
```

Docker Compose автоматически:
1. Запустит Qdrant и дождётся его готовности (healthcheck)
2. Запустит бэкенд, который проверит все зависимости
3. Запустит фронтенд после готовности бэкенда

- **Фронтенд:** http://localhost:3000
- **Бэкенд:** http://localhost:8000
- **Qdrant Dashboard:** http://localhost:6333/dashboard
- **API-документация:** http://localhost:8000/docs

#### 4. Проверка работоспособности

```bash
curl http://localhost:8000/health
```

Ожидаемый ответ:
```json
{
  "status": "healthy",
  "qdrant": "ok",
  "yandex_api": "ok",
  "alice": "ok"
}
```

### Вариант B: Локальный запуск (для разработки)

#### 1. Клонирование и настройка

```bash
git clone <repo-url>
cd Alice_Hackaton
cp .env.example .env
# Отредактируйте .env, указав реальные ключи
# При локальном запуске QDRANT_URL=http://localhost:6333
```

#### 2. Запуск Qdrant

```bash
docker run -d --name qdrant \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant_data:/qdrant/storage \
  qdrant/qdrant:latest
```

#### 3. Запуск бэкенда

```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# venv\Scripts\activate   # Windows

pip install -r requirements.txt
uvicorn backend.main:app --reload --port 8000
```

> **Важно:** Если бэкенд не запускается — проверьте логи. Приложение выведет подробную информацию о том, какая зависимость недоступна и как это исправить.

Проверка: http://localhost:8000/health

#### 4. Запуск фронтенда

```bash
cd frontend
npm install
npm run dev
```

Фронтенд запустится на http://localhost:3000

## Устранение ошибок запуска

### Ошибка: «Qdrant недоступен»

```
✗ Qdrant недоступен: Не удалось подключиться к Qdrant по адресу http://localhost:6333
```

**Решение:**
- Убедитесь, что Qdrant запущен: `docker ps | grep qdrant`
- Проверьте `QDRANT_URL` в `.env`:
  - Docker Compose: `http://qdrant:6333`
  - Локальный запуск: `http://localhost:6333`
- Запустите Qdrant: `docker run -d -p 6333:6333 qdrant/qdrant:latest`

### Ошибка: «Yandex API недоступен»

```
✗ YANDEX_API_KEY не задан в переменных окружения
```

**Решение:**
- Получите API-ключ в [Yandex Cloud Console](https://console.yandex.cloud/)
- Убедитесь, что каталог создан и API активированы
- Установите `YANDEX_API_KEY` и `YANDEX_FOLDER_ID` в `.env`

### Предупреждение: «ALICE_SKILL_ID и/или ALICE_OAUTH_TOKEN не заданы»

```
⚠ ALICE_SKILL_ID и/или ALICE_OAUTH_TOKEN не заданы
```

Это **предупреждение, а не ошибка** — приложение продолжит работу. Вебхук Алисы (`/api/v1/alice/webhook`) будет доступен, но управление навыком через API будет недоступно.

**Для полной интеграции:**
- Создайте навык в [Яндекс.Диалогах](https://dialogs.yandex.ru/)
- Скопируйте Skill ID и OAuth Token
- Установите `ALICE_SKILL_ID` и `ALICE_OAUTH_TOKEN` в `.env`

## Пользовательские сценарии

### Сценарий 1: Создание теста

1. Пользователь регистрируется / входит в систему
2. Нажимает «Создать тест»
3. Вводит название и текст теста (или загружает PDF/DOCX)
4. Выбирает тип адаптации (без адаптации / слабовидение / когнитивные / ОДА)
5. Нажимает «Создать тест»
6. Получает готовый тест с ссылкой для обмена

### Сценарий 2: Прохождение теста

1. Пользователь открывает тест из списка или по ссылке
2. Отвечает на вопросы, выбирая варианты
3. Алиса помогает подсказками при необходимости
4. После завершения видит результат в процентах

### Сценарий 3: Доступ по ссылке

1. Автор копирует ссылку на тест
2. Отправляет ссылку другому пользователю
3. Получатель переходит по ссылке и сразу начинает тест

### Сценарий 4: Голосовой интерфейс Алисы

1. Пользователь активирует навык Алисы: «Алиса, запусти навык Доступное Обучение»
2. Алиса приветствует и предлагает: тест, подсказка, адаптация
3. Пользователь голосом выбирает действие
4. Алиса ведёт диалог, зачитывает вопросы, принимает ответы

### Сценарий 5: Голосовой помощник на фронтенде для слабовидящих

1. Пользователь открывает тест на платформе
2. Активирует голосовой режим (автоматически или вручную)
3. Система приветствует и объясняет доступные команды
4. Пользователь говорит "начать"
5. Система зачитывает первый вопрос и варианты ответов
6. Пользователь отвечает голосом ("А", "Б", "В", "Г")
7. Система подтверждает ответ и переходит к следующему вопросу
8. При необходимости пользователь может запросить подсказку, повторить вопрос или завершить тест

## Ограничения

- In-memory хранилище данных (при перезапуске бэкенда данные сбрасываются)
- Все зависимости (Qdrant, Yandex API, Алиса) обязательны для работы
- Голосовой интерфейс через Алису требует настройки навыка в Яндекс.Диалогах и публичного URL для вебхука

## Перспективы развития

- Переход на PostgreSQL для постоянного хранения данных
- Полная интеграция с навыком Алисы через Яндекс.Диалоги
- MCP-интеграции для подключения внешних образовательных платформ
- Мультимодальность: обработка аудио/видео контента
- Расширенная аналитика для педагогов
- Поддержка IETF-форматов тестов (QTI, GIFT)
